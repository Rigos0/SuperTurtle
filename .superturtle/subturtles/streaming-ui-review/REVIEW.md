# Telegram Streaming UI Review

## Findings

1. P1: Codex can abort valid long-running tool work 120 seconds earlier than Claude. `CodexSession.sendMessage()` always races the event iterator against `EVENT_STREAM_STALL_TIMEOUT_MS`, even after a tool has started, so a quiet MCP tool run is treated as a stall and aborted after the normal stream timeout. Claude switches to `TOOL_ACTIVE_STALL_TIMEOUT_MS` once a `tool_use` block starts, which gives tool execution a longer grace window before teardown. This is a real parity regression for long-running tool calls with sparse progress events. Refs: `super_turtle/claude-telegram-bot/src/codex-session.ts:1310-1338`, `super_turtle/claude-telegram-bot/src/session.ts:722-800`.
2. P2: Codex adds retained-progress tool snapshots for UI-owned MCP tools that Claude intentionally suppresses. On every completed `mcp_tool_call`, Codex emits `statusCallback("tool", ...)` before handling side effects, including `ask_user`, `send_image`, `send_turtle`, `bot_control`, and `pino_logs`. Claude explicitly filters those tool names out because they already render their own Telegram UI or side effects. The result is different progress text and extra "Using tools" history entries for equivalent user flows. Refs: `super_turtle/claude-telegram-bot/src/codex-session.ts:1417-1436`, `super_turtle/claude-telegram-bot/src/session.ts:846-879`, `super_turtle/claude-telegram-bot/src/message-kinds.ts:30-44`.
3. P3: Codex hardcodes streamed-text throttling to `500` ms instead of using the shared config constant. Claude reads `STREAMING_THROTTLE_MS`, while Codex compares `now - lastTextUpdate > 500`. Behavior matches today only because the config constant currently equals `500`; any future config change will silently desynchronize Claude and Codex pacing. Refs: `super_turtle/claude-telegram-bot/src/codex-session.ts:1397-1401`, `super_turtle/claude-telegram-bot/src/session.ts:980-991`, `super_turtle/claude-telegram-bot/src/config.ts:489-491`.

## Parity Confirmed

- Retained-progress teardown ordering matches. `CodexDriver.runMessage()` defers downstream `done` until its pending-output pump stops and the final flush finishes, which preserves the same "deliver side effects before retained-progress completion" behavior Claude gets by handling pending outputs inline before `done`. Refs: `super_turtle/claude-telegram-bot/src/drivers/codex-driver.ts:16-89`, `super_turtle/claude-telegram-bot/src/session.ts:881-972`, `super_turtle/claude-telegram-bot/src/session.ts:1104-1116`.
- Session persistence timing matches. Codex saves as soon as `thread_id` arrives and again after finalizing the assistant reply; Claude does the same when `session_id` arrives and after persisting the completed response buffer. Refs: `super_turtle/claude-telegram-bot/src/codex-session.ts:1352-1367`, `super_turtle/claude-telegram-bot/src/codex-session.ts:1566-1573`, `super_turtle/claude-telegram-bot/src/session.ts:776-780`, `super_turtle/claude-telegram-bot/src/session.ts:1111-1116`.
- Arrow navigation behavior is shared across both drivers. `handleCallback()` routes `progress_nav:*` into `navigateRetainedProgressViewer()`, and the viewer lookup is keyed only by `chat_id + message_id`, so both Claude and Codex get the same boundary and missing-history behavior. Refs: `super_turtle/claude-telegram-bot/src/handlers/callback.ts:436-448`, `super_turtle/claude-telegram-bot/src/handlers/streaming.ts:958-985`.

## Test Gaps

- The focused tests cover the shared renderer and callback routing, including silent placeholder creation, thinking/tool updates, bounded snapshot history, navigation controls, minimum on-screen pacing, and final artifact promotion. Refs: `super_turtle/claude-telegram-bot/src/handlers/streaming.test.ts:540-1069`, `super_turtle/claude-telegram-bot/src/handlers/callback.test.ts:65-230`.
- There is still no direct regression test that executes the Claude or Codex streaming loops themselves, so the three parity deltas above are currently unguarded. Refs: `super_turtle/claude-telegram-bot/src/codex-session.ts:1310-1436`, `super_turtle/claude-telegram-bot/src/session.ts:722-991`.
